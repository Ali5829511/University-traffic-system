نظرة عامة

يدعم نظام إدارة المرور الجامعي النشر كحاوية Docker، مما يتيح لك تشغيل النظام بسهولة على أي منصة سحابية (Azure, AWS, GCP) أو على خادم داخلي.

PostgreSQL مع تهيئة تلقائية

البدء السريع

لتشغيل النظام كاملاً مع قاعدة البيانات بأمر واحد:

# تشغيل النظام مع قاعدة البيانات
docker compose up -d

# عرض السجلات
docker compose logs -f

# إيقاف النظام
docker compose down

بعد التشغيل، يمكنك الوصول للنظام على: http://localhost:3000

خطوات النشر

  • تثبيت Docker تأكد من تثبيت Docker و Docker Compose على الخادم
  • نسخ المشروع انسخ ملفات المشروع إلى الخادم أو استخدم git clone
  • إعداد متغيرات البيئة قم بتعديل كلمات المرور الافتراضية في ملف .env
  • تشغيل Docker Compose شغّل الأمر docker compose up -d لبدء النظام
  • التحقق من التشغيل افتح http://localhost:3000 للتأكد من عمل النظام

متغيرات البيئة

يمكنك تخصيص إعدادات النظام من خلال متغيرات البيئة:

المتغير الوصف القيمة الافتراضية
DB_USER اسم مستخدم قاعدة البيانات traffic_user
DB_PASSWORD كلمة مرور قاعدة البيانات traffic_password
DB_NAME اسم قاعدة البيانات traffic_system
PORT منفذ التطبيق 3000
DB_PORT منفذ قاعدة البيانات 5432 
# تشغيل مع إعدادات مخصصة
DB_PASSWORD=your_secure_password docker compose up -d

تحذير أمني مهم

تأكد من تغيير كلمات المرور الافتراضية قبل النشر في بيئة الإنتاج! استخدم كلمات مرور قوية ولا تشارك ملف .env مع أي شخص.

أوامر متقدمة 

# بناء الصورة بشكل منفصل
docker build -t university-traffic-system .

# تشغيل مع قاعدة بيانات خارجية
docker run -d \
  -e DATABASE_URL=postgresql://user:pass@host:5432/db \
  -e DB_SSL=true \
  -p 3000:3000 \
  university-traffic-system

# عرض حالة الحاويات
docker compose ps

# إعادة بناء الصورة
docker compose build --no-cache

# حذف البيانات وإعادة البدء
docker compose down -v
docker compose up -d
